## Enterprise Control Layers

1. Input sanitation and prompt-injection screening
2. Runtime policy checks before sensitive tools
3. Output moderation and format enforcement
4. Risk-tiered human approvals
5. Incident logging and evidence retention

<RoleCallout role="dev">
Guardrails belong outside prompts too. Treat policy checks as deterministic code in the runtime path.
</RoleCallout>

<RoleCallout role="qa">
Create adversarial suites for prompt leak attempts and cross-tenant data exfiltration.
</RoleCallout>

<RoleCallout role="pm">
Define “critical action” taxonomy with legal/compliance stakeholders before launch.
</RoleCallout>

<Callout type="warning" title="Production Gotcha">
If governance is optional by feature flag, someone will disable it under delivery pressure. Make core guardrails non-bypassable for high-risk paths.
</Callout>
