## Protocols Over Frameworks

Interoperability comes from stable contracts, not shared libraries.

  title="A2A-Oriented Multi-Agent Topology"

## Contract Essentials

- Task envelope schema
- Capability advertisement
- Timeout and cancellation semantics
- Error taxonomy
- Trace correlation IDs

<RoleCallout role="dev">
Define protocol schemas first, then adapter layers per framework. This prevents framework lock-in.
</RoleCallout>

<RoleCallout role="qa">
Interoperability tests should include mixed-version agents and partial outages.
</RoleCallout>

<RoleCallout role="pm">
Use capability contracts to decide ownership boundaries across teams and vendors.
</RoleCallout>

<Callout type="tip" title="Migration Strategy">
Start with one delegated domain flow and enforce contract compatibility in CI before broad rollout.
</Callout>
