## Memory is a System, Not a Feature

Enterprise agents need multiple memory types with explicit retention and retrieval rules.

## Practical Budgeting Model

- Reserve tokens for policy/system instructions.
- Reserve tokens for tool outputs.
- Allocate dynamic budget for retrieval chunks.
- Persist summaries, not raw transcript dumps.

<RoleCallout role="dev">
Introduce a context assembler layer that enforces budgets by policy. Don’t let arbitrary callers append context directly.
</RoleCallout>

<RoleCallout role="qa">
Run controlled tests with stale memory records to ensure the agent prefers fresh, trusted retrieval when conflicts appear.
</RoleCallout>

<RoleCallout role="pm">
Set memory retention windows by user impact and compliance needs. Infinite retention is usually a liability.
</RoleCallout>

<Callout type="tip" title="Cost Lever">
Prompt caching and reusable system blocks often reduce cost more than switching to a cheaper model tier.
</Callout>
